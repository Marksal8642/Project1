**script.js:**
```js
const users = [
  { username: 'user1', password: 'password1' },
  { username: 'user2', password: 'password2' }
];

const loginForm = document.getElementById('loginForm');
const messageElement = document.getElementById('message');

loginForm.addEventListener('submit', function (event) {
  event.preventDefault();
  const formData = new FormData(loginForm);
  const enteredUsername = formData.get('username');
  const enteredPassword = formData.get('password');

  // Basic client-side validation
  if (!enteredUsername || !enteredPassword) {
    messageElement.textContent = 'Please enter both username and password.';
    return;
  }

  const user = users.find(u => u.username === enteredUsername && u.password === enteredPassword);

  if (user) {
    messageElement.textContent = `Welcome, ${enteredUsername}!`;
    // Here, you can redirect the user to another page after successful login.
    // For this example, let's just display a welcome message.
  } else {
    messageElement.textContent = 'Invalid username or password.';
  }
});
```

In this updated version, we have added a basic client-side validation check to ensure that both the username and password fields are filled before attempting to log in. If either of the fields is empty, an error message will be displayed, prompting the user to fill in both fields.

Again, remember that client-side validation is not enough to guarantee security. Server-side validation is crucial to protect your application from attacks and ensure the security of user credentials. When building a real login system, you must implement server-side validation to verify the credentials against the database securely.

**script.js:**
```js
const users = [
  { username: 'user1', password: 'password1' },
  { username: 'user2', password: 'password2' }
];

const loginForm = document.getElementById('loginForm');
const messageElement = document.getElementById('message');
const usernameInput = document.getElementById('username');
const passwordInput = document.getElementById('password');
const usernameError = document.getElementById('usernameError');
const passwordError = document.getElementById('passwordError');

loginForm.addEventListener('submit', function (event) {
  event.preventDefault();
  clearErrors();

  const enteredUsername = usernameInput.value.trim();
  const enteredPassword = passwordInput.value;

  // Check for empty fields
  if (!enteredUsername || !enteredPassword) {
    if (!enteredUsername) {
      displayError(usernameError, 'Please enter your username.');
    }
    if (!enteredPassword) {
      displayError(passwordError, 'Please enter your password.');
    }
    return;
  }

  // Check if username exists in the mock database
  const user = users.find(u => u.username === enteredUsername);

  if (!user) {
    displayError(usernameError, 'Invalid username or password.');
    return;
  }

  // Validate the password
  if (user.password !== enteredPassword) {
    displayError(passwordError, 'Invalid username or password.');
    return;
  }

  // Login successful
  displaySuccessMessage(`Welcome, ${enteredUsername}!`);
});

function clearErrors() {
  usernameError.textContent = '';
  passwordError.textContent = '';
  messageElement.textContent = '';
}

function displayError(errorElement, message) {
  errorElement.textContent = message;
}

function displaySuccessMessage(message) {
  messageElement.textContent = message;
  messageElement.style.color = 'green';
}
```

This example includes the following client-side validation features:

1. **Empty Field Check:** It checks whether both the username and password fields are filled. If either of them is empty, it displays an appropriate error message below the input field.

2. **Username Existence Check:** It verifies whether the entered username exists in the mock database. If not, an error message is shown to indicate an invalid username or password.

3. **Password Validation:** After finding the user with the entered username, the script checks if the entered password matches the password associated with the username. If they don't match, an error message is displayed.

4. **Error Messages:** Error messages are displayed in red below the respective input fields when validation fails. A success message is displayed in green if the login is successful.

Remember that this client-side validation is for user experience only and should be complemented with server-side validation for security purposes in a real-world application. Additionally, in a production environment, you should avoid storing passwords in plain text and consider using secure hashing algorithms to protect user data.
